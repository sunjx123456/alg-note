### DAG的遍历
遍历是指数复杂度的，因为不是所有只遍历一次，因此这道题需要把记忆化把遍历变成O(n)
### 细节问题
要先把操作一处理完毕才能改标记，等等
## 优化
### 取模运算的优化
int add(int a, int b)
{
	return (a + b >= mod) ? (a + b - mod) : a + b;
}

有一定效果但不一定明显，在不同平台不一样，考试平台快了200ms，但是清研没有效果
### map和unordered_map
map只有需要排序的情况下用，这里显然只是需要一个字符串的映射，应该用unordered_map

效果显著的优化(400ms)
### string vector
尽量少初始化声明，耗时，用全局变量，vector宁愿clear也别重新初始化，clear永远都比重新快

效果也很显著(200ms)
### 题目的中的优化
动态字符串时如果不求值只需要把长度记录下就行了，不用放到vector里
 
 效果显著
 ### const &
 像需要table[x]这种，直接初始化依旧耗费巨大，直接调用反而不会太多，但是最好的是const vector\<string>\&tmp

 效果依旧显著
